# SCALE.md

Standardised scales used throughout the training log.

---

## Pain / Injury (1–10)

Custom scale combining intensity and functional impact. Not the clinical NRS-11 (which is pure intensity: 0 = no pain, 10 = worst imaginable) — this scale is anchored to training-relevant thresholds.

| Score | Description |
|-------|-------------|
| 0 | No pain |
| 1 | Noticeable only when concentrating on it |
| 2 | Noticeable when active, not distracting |
| 3 | Mildly annoying when active, no performance impact |
| 4 | Pain when active, stays the same or improves with movement, full range of motion |
| 5 | Pain when active, does not improve with movement |
| 6 | Pain present even at rest, manageable |
| 7 | Significant but tolerable pain when active |
| 8 | Significant pain at rest |
| 9 | Significant pain when active, movement restricted |
| 10 | Significant pain even at rest, unable to train |

---

## Effort — RPE (1–10)

Standard Borg CR-10 rate of perceived exertion.

| Score | Description |
|-------|-------------|
| 1 | Rest / barely any effort |
| 2 | Easy, could do this all day |
| 3 | Comfortable, breathing slightly elevated |
| 4 | Moderate, starting to breathe harder |
| 5 | Challenging, conversation in short sentences |
| 6 | Hard, would rather stop but manageable |
| 7 | Very hard, short answers only |
| 8 | Extremely hard, can sustain for limited time |
| 9 | Near max, almost nothing left |
| 10 | Max effort, cannot continue |

---

## Execution (1–10)

Combined assessment of technical quality and effort management.

| Score | Description |
|-------|-------------|
| 1 | Everything went wrong — pacing, technique, decisions |
| 2 | Major errors across multiple areas |
| 3 | Poor execution, some things worked but mostly off |
| 4 | Below average — noticeable pacing or technique issues |
| 5 | Average — nothing special, some minor mistakes |
| 6 | Solid — executed the plan with minor deviations |
| 7 | Good — smart pacing, clean technique, few errors |
| 8 | Very good — near-optimal decisions and movement quality |
| 9 | Excellent — couldn't have done much better on the day |
| 10 | Perfect — technically flawless, optimal effort management |
